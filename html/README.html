RL Foundation5 RL Basics 分类一句话通俗理解 ：从 MDP 到 GAE 的最小闭环，把“如何定义问题、如何估值、如何降方差”串成一条可复现学习路径。 Advantage（优势函数学习与对比）一句话通俗理解 ：Advantage 用“动作回报 - 状态基线”衡量决策增益，让策略更新聚焦真正带来提升的动作。 GAE（Generalized Advantage Estimation）一句话通俗理解 ：GAE 通过指数加权的多步 TD 残差在偏差与方差间折中，让策略梯度既稳又不迟钝。 MDP（马尔可夫决策过程）一句话通俗理解 ：MDP 用状态、动作、转移、奖励、折扣把决策问题形式化，为价值迭代与策略优化提供统一语义。 TD Learning（时序差分学习）一句话通俗理解 ：TD Learning 边交互边自举更新价值估计，不等回合结束也能持续学习并加速收敛。
Architecture10 Architecture 分类一句话通俗理解 ：架构模块回答三个核心问题: 模型怎么“看懂上下文”、怎么“跨模态理解”、怎么“高效生成”。 Diffusion（扩散模型）一句话通俗理解 ：Diffusion 先学会逐步加噪再反向去噪，把生成任务转化为一系列可控的噪声还原步骤。 DiT（Diffusion Transformer）一句话通俗理解 ：DiT 用 Transformer 替代 U-Net 处理扩散过程中的 token 化图像表示，在规模化训练下提升生成质量。 Generation & Decoding (推理与生成优化)一句话通俗理解 ：生成优化是在质量不明显下降的前提下，把“首字延迟、吞吐、显存”三件事同时做平衡。 Transformer Core (大语言模型架构)一句话通俗理解 ：Transformer 通过多头注意力与前馈残差堆叠，将离散 token 编码为可推理的上下文表示并支持自回归生成。 BLIP2一句话通俗理解 ：BLIP-2 用 Q-Former 把视觉特征压缩成语言模型可消费的表示，在少量训练下实现图文能力迁移。 Flamingo一句话通俗理解 ：Flamingo 在冻结大模型前提下插入跨模态注意力层，让语言模型按需读取视觉上下文完成多模态对话。 LLaVA一句话通俗理解 ：LLaVA 通过视觉编码器与 MLP 投影把图像 token 对齐到 LLM 语义空间，实现低成本视觉指令跟随。 多模态 VLM (Vision-Language Models)一句话通俗理解 ：VLM 的关键是把图像特征“翻译”为语言模型能消费的 token，再让同一套生成机制完成理解和回答。 Transformer 注意力机制 (Attention Mechanisms)一句话通俗理解 ：注意力机制通过 Q-K 匹配动态分配信息流，GQA/MQA 与 FlashAttention 在精度与成本间给出不同工程折中。
Alignment11 Alignment 分类一句话通俗理解 ：对齐阶段把“会续写”转成“会按人类意图回答”，核心是数据质量、偏好信号和策略约束的联动优化。 Actor-Critic一句话通俗理解 ：Actor-Critic 让策略网络负责行动、价值网络负责纠偏，用低方差优势信号提升学习稳定性。 数据合成 (Data Synthesis)一句话通俗理解 ：数据合成用模型自举扩充训练样本，再借助规则或验证器筛选高质量数据以降低标注成本。 核心问题 ：当真实标注数据稀缺、昂贵或存在隐私风险时，如何用程序或 LLM… DPO（直接偏好优化）一句话通俗理解 ：DPO 直接在偏好对上优化策略相对概率，无需单独训练奖励模型即可实现稳定对齐。 GRPO (Group Relative Policy Optimization) 组内相对策略优化一句话通俗理解 ：GRPO 在同一问题内做组内相对比较，利用标准化优势信号提升推理对齐的样本效率与稳定性。 高效微调 (PEFT: Parameter-Efficient Fine-Tuning)一句话通俗理解 ：PEFT 只更新少量可训练参数（如 LoRA），在显存可控的前提下快速迁移模型能力。 Policy Gradient（策略梯度）一句话通俗理解 ：Policy Gradient 直接最大化期望回报，对离散决策友好但需借助基线与技巧控制方差。 PPO (Proximal Policy Optimization) 强化学习对齐一句话通俗理解 ：PPO 用裁剪目标限制新旧策略偏移，在可学习性与训练稳定性之间取得工程上最实用的平衡。 RLHF (Reinforcement Learning from Human Feedback) 人类反馈强化学习一句话通俗理解 ：RLHF 通过 SFT、奖励建模和策略优化三段式闭环，把人类偏好显式注入模型行为。 SFT (Supervised Fine-Tuning) 监督微调一句话通俗理解 ：SFT 用高质量指令-响应样本做监督学习，先建立可控表达基线再进入偏好对齐阶段。 数据工程与模型评估 (Data & Evaluation)一句话通俗理解 ：数据工程通过清洗、配比、去重和评测闭环决定对齐上限，是比模型结构更常见的效果瓶颈。
Advanced Topics4 Advanced Topics 分类一句话通俗理解 ：进阶模块聚焦“标准配方之外”的难题，核心是用更强约束解决更高风险场景。 BCQ（Batch-Constrained Q-learning）一句话通俗理解 ：BCQ 把策略限制在行为数据支持的动作邻域内，减少离线场景中分布外动作带来的价值幻觉。 CQL（Conservative Q-Learning）一句话通俗理解 ：CQL 通过保守正则主动压低未见动作 Q 值，降低离线强化学习的过估计风险。 Offline RL 分类一句话通俗理解 ：Offline RL 在不与环境交互的条件下仅凭历史轨迹学习策略，关键挑战是分布偏移与价值过估计。
Engineering6 Systems 分类一句话通俗理解 ：系统工程通过并行、精度、算子与调度协同优化，把大模型训练推理从“能跑”提升到“高效稳定地跑”。 CUDA 专题一句话通俗理解 ：CUDA 通过显式控制线程、内存层次与核函数执行路径，把 GPU 吞吐潜力转化为实际训练速度。 DeepSpeed 专题一句话通俗理解 ：DeepSpeed 以 ZeRO 为核心重构状态存储与通信开销，使超大模型在有限显存下可训练。 推理优化 (Inference Optimization)一句话通俗理解 ：推理优化围绕 KV Cache、批调度与算子融合，目标是在延迟、吞吐与成本之间找到最优点。 Megatron 专题一句话通俗理解 ：Megatron-LM 通过 TP/PP/DP 组合切分模型与计算图，让超大参数训练跨多机多卡高效扩展。 混合精度训练（Mixed Precision）一句话通俗理解 ：混合精度在保持数值稳定的前提下用低精度计算主路径，并通过 loss scaling 避免梯度下溢。
Agent6 Agentic Reasoning (智能体推理)一句话通俗理解 ：Agent 把语言模型升级为可执行系统：先规划、再调用工具、再反思修正，直到任务完成。 Agent Frameworks（常见框架与示例代码）一句话通俗理解 ：Agent 框架是把“模型 + 工具 + 状态 + 编排”打包成可维护工程系统的脚手架。 记忆与检索 (Memory & RAG)一句话通俗理解 ：Memory 与 RAG 把短期上下文和长期检索结合，让 Agent 在长任务里保持一致性与可追溯性。 多智能体系统 (Multi-Agent Systems)一句话通俗理解 ：多智能体通过角色分工与协作协议并行处理子任务，提升复杂任务的鲁棒性与执行效率。 OpenClaw 架构深度解析一句话通俗理解 ：OpenClaw 以本地优先运行时和文件记忆体系实现可控自动化，强调可审计、可恢复与低耦合。 OpenClaw 是一个开源的本地优先（Local-First… 智能体编排范式 (Agent Orchestration)一句话通俗理解 ：编排层定义 Agent 的决策流程与状态转移规则，决定系统能否在复杂场景中稳定推进。

🧠 核心技术参考 (Technical Reference)#

1. 显存计算与容量估算 (Memory & Compute)#

  • 静态权重：fp16 占 2 Bytes/Param。
  • KV Cache（总量）：batch_size × seq_len × 2 × layers × kv_heads × head_dim × precision_bytes。
    （其中 2 表示 K/V 两份缓存）
  • 量化增益：通过 定点量化 (INT4/INT8)，显存占用可降低 50%-75%。

机制 特点 显存优化

MQA Multi-Query Attention KV 头=1
GQA Grouped-Query Attention 折中方案

3. 对齐算法演进#

阶段 算法 核心思想

DPO Direct Preference Optimization 离线对比优化
GRPO Group Relative Policy Optimization 组内相对优势

4. Agent 架构演进#

  • ReAct 范式：协同推理（Reason）与行动（Act），动态调整计划。
  • Plan and Execute：先计划再执行，适合长链条任务。
  • Multi-Agent Mesh：去中心化编排，支持分布式决策与角色分担。

5. 分布式训练策略#

  • 数据并行 (DP)：副本间切分数据，All-Reduce 同步梯度
  • 张量并行 (TP)：切分层内权重，All-Gather 激活
  • 流水线并行 (PP)：按层切分 Stage，Bubble 时间
  • 专家并行 (EP)：MoE 特有，All-to-All 路由专家
